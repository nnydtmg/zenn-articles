# ファクトチェック スキル

指定された記事ファイルを読み込み、技術的事実・サービス仕様・コードの正確性を検証する。
検証には WebSearch / WebFetch を積極的に使い、思い込みで判断しない。

## チェック対象カテゴリ

### A. AWSサービス・機能名
- サービス名の正式表記（例: "Amazon CloudWatch" vs "AWS CloudWatch"）
- 機能名の正確性（例: "Transaction Search" の正式名称）
- GA/Preview 状態（記事中の「プレビュー」「GA」が現時点で正しいか）

### B. バージョン・リリース日
- SDK バージョン番号
- ランタイムバージョン（例: `syn-nodejs-puppeteer-11.0` など）
- 「〇〇年〇月にリリース」等の日付記述
- 現在の最新バージョンと記事記載バージョンのギャップ

### C. OTLP / API エンドポイント・パラメータ
- エンドポイント URL の形式と正確性
- 環境変数名・設定キー名の正確性（例: `OTEL_EXPORTER_OTLP_TRACES_ENDPOINT`）
- 対応プロトコル（gRPC / HTTP 等）の記述

### D. IAM ポリシー・マネージドポリシー名
- `AWSXRayDaemonWriteAccess` 等のマネージドポリシー名が実在するか
- アクション名（`rum:PutRumEvents` 等）の正確性

### E. 制限値・コスト
- リクエストサイズ制限（例: "1リクエスト最大 5MB"）
- Free Tier 条件（例: "月間100実行以上はFree Tierを外れる"）
- リージョン別の差異

### F. コードサンプル
- 構文エラーがないか
- ライブラリの import / require が記事の文脈と整合するか
- 記事内の複数コードブロック間での変数名・設定値の整合性

### G. 外部リンク・URL（形式の確認のみ）
- 文中に記載されたURL形式が適切か（壊れたリンクの兆候）
- 参照先の記事タイトルと URL が合致しているか（タイトルで確認可能な範囲）

## 実行手順

### 1. 対象ファイルの特定
ユーザーがファイルパスを指定していない場合は確認する。デフォルトは直近の会話で話題になっているファイル。

### 2. ファイルを読み込む
```
Read(file_path)
```

### 3. 要検証クレームを抽出する
記事を通読し、上記カテゴリに該当する技術的クレームをリストアップする。
明らかに正しいものはスキップ。「確信が持てない」「古い可能性がある」クレームを優先する。

### 4. WebSearch / WebFetch で裏取りする

各クレームについて:
```
WebSearch("AWS <サービス名> <機能名> <キーワード> site:docs.aws.amazon.com OR aws.amazon.com")
WebFetch(url) # 公式ドキュメントページを直接確認
```

優先する情報源（信頼度順）:
1. `docs.aws.amazon.com` — AWS公式ドキュメント
2. `aws.amazon.com/about-aws/whats-new/` — AWS What's New（リリース日確認）
3. `aws.amazon.com/blogs/` — AWS公式ブログ
4. `opentelemetry.io/docs/` — OTel公式ドキュメント（OTel関連の場合）

### 5. 結果を報告する

以下のフォーマットで出力:

```
## ファクトチェック結果: <ファイル名>

### サマリ
<全体的な評価（1〜2文）>
- 確認済みクレーム: X件（正確）
- 要確認/修正: Y件
- 確認不能（情報源不足）: Z件

---

### 要修正・要確認

**[問題1]** 行番号: XX
- カテゴリ: <A〜Gのどれか>
- 記載内容: 「<記事内のテキスト>」
- 確認結果: <何が問題か>
- 根拠: <参照URL または情報源>
- 修正案: <正しい記述>

（繰り返し）

---

### 確認済み（正確）
- 行XX: 「<クレーム>」→ 正確（参照: <URL>）
（主要なクレームのみ記載）

---

### 確認不能
- 行XX: 「<クレーム>」→ <確認できなかった理由>
  → 推奨: 公式ドキュメントで手動確認してください
```

### 6. 修正を適用する場合

ユーザーから「修正して」と言われたら、指摘箇所を実際に Edit ツールで修正する。
事実の修正のみを行い、**文体・トーンは変えない**。

## 注意事項

- AWSサービスは頻繁にアップデートされるため、「記事執筆時点では正しかった」可能性がある。執筆日（frontmatterのupdated_atやコミット履歴）も参照する
- コードサンプルの構文チェックは静的解析レベルで行う。実行検証は行わない
- 「〜と思います」等の主観的表現はチェック対象外
- 価格情報は変動が激しいため「要手動確認」として報告する
